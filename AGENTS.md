## 세션 연속성

사용자가 세션 시작 시 아래와 같은 뉘앙스의 말을 하면, **반드시** `.ai/local/STATE.md` 파일을 읽고 현재 진행 상태를 파악한 뒤 답변하라.

트리거 예시: "이전 내용 보고 와", "지난 대화 이어서", "전에 어디까지 했지", "이어서 하자", "연속성 유지해줘", "전 세션 내용 참고해", 그 외 이전 맥락을 요청하는 모든 표현.

읽은 후에는 STATE 기준으로 현재 진행 상태(완료된 것 / 남은 것)를 간략히 요약하고, 이어서 작업을 진행하라.

## 완료 기록 (WORKLOG)

이슈/기능 단위 작업이 완료되면 **반드시** `.ai/local/WORKLOG.md`에 기록을 추가하라.

- **시점**: "완료/끝났어/다음으로 가자" 등 완료 의사 표시, PR 생성 직후, 기능 동작 확인 후
- **항목**: 날짜·이슈번호·브랜치·상태 / 변경 파일(생성·수정·삭제 구분) / 주요 작업 내용 / 트러블슈팅(있으면) / 부수 결정(있으면)
- 최신 기록이 위로 오도록 **위에 쌓는다**. 진행 중 작업은 `🚧 진행 중` stub만 미리 만들어도 OK.

## 도메인 작업 규칙

특정 도메인의 코드를 작업할 때는 **먼저 그 도메인 `README.md`(역할·용어)와 루트 `CONTEXT-MAP.md`(경계·의존 방향)를 읽어라.** 경계를 넘는 호출은 포트/이벤트로만 한다(직접 SpringData 참조 금지 → `docs/adr/0003-bc-communication.md`).

## API 작업 규칙

Controller/Service/DTO/Repository 작성·수정, 엔드포인트 추가·변경, Request/Response·에러코드 변경, "API 만들어줘" 등 언급 시 **반드시 먼저 `.ai/local/API.md`를 읽어라.** 명세와 다른 경로/메서드/필드를 쓰지 말 것. 명세가 바뀌면 `API.md`도 함께 갱신하라.

## STATE 동기화

토큰 절약을 위해 변경 즉시 반영하지 말고 **이슈 하나 완료 시점 또는 대화 마무리 지점**에 모아서 `.ai/local/STATE.md`에 반영하라.

- **대상**: 도메인 설계 변경(ERD·엔티티·역할), 정책 변경, 액션아이템 완료(`[ ]`→`[x]`), 마일스톤, 설계 결정
- **방식**: "반영했습니다" 별도 고지 없이 **조용히 처리**, 기존 포맷·스타일 유지, 변경일자 있으면 `⚠️ 변경사항` 섹션에 날짜와 함께 추가
- 백로그는 STATE의 `🔖 백로그` 섹션에 등록일자와 함께 추가

## 스킬 (절차형 작업)

`.claude/skills/` 에 팀 공용 절차 스킬이 있다. **Claude Code**는 네이티브 스킬로 자동 인식·트리거한다.

- **Codex 등 비-Claude 도구**: 절차형 작업 요청이 오면 `.claude/skills/` 에 해당 스킬이 있는지 먼저 보고, 있으면 그 `SKILL.md` **본문 절차를 따르라**(frontmatter는 무시, 본문만). 새 스킬이 추가돼도 이 규칙으로 자가 발견한다.
- 현재 스킬: `grill-me` — 계획/설계를 1문1답으로 끝까지 그릴(검증). "그릴해줘", "이 plan 검증해줘" 등에서 발동.
- ⚠️ 스크립트·MCP 의존 스킬은 **Claude 전용**이다. 비-Claude 도구는 따르지 말 것.

## 응답 스타일

- **응답 형식**: `docs/response-style.md`를 따른다(팀 공용 — 톤·코드 제공·단계 분해·트러블슈팅 양식 등).
- **PR·커밋 양식**: `docs/PR-COMMIT.md`를 따른다(팀 합의 양식).

매번 적용하는 **불변 규칙**(이것만 항상 지킨다):

- **한국어**로 답한다.
- **결론 먼저**, 설명 나중. 한 응답에 너무 많은 단계를 묶지 말고 한 번에 한 단계씩.
- 코드만 던지지 말고 **경로·설명**을 함께. 매 코드 변경 후 "빌드 확인" 권유.
- 🚫 **커밋·푸시·PR 직접 실행 금지** (`git commit` / `git push` / `gh pr create` 등). 사용자가 직접 한다. AI는 양식대로 **텍스트만** 제공한다.
