## M4 목표 및 기대 결과

- 관리자가 이미지와 지급 조건을 포함한 뱃지를 등록, 조회, 수정, 삭제할 수 있다.
- 학생이 누적 포인트 조건을 만족한 뱃지를 자동으로 획득할 수 있다.
- 학생이 획득한 뱃지 중 대표 뱃지를 선택할 수 있다.
- 뱃지 이미지가 GCS에 저장되고 DB에는 이미지 URL과 메타데이터가 저장된다.
- API가 빈 성공 응답이나 `null` 대신 실제 데이터 또는 명확한 도메인 예외를 반환한다.
- 문제 또는 문제 세트 기준으로 연관 강의/강좌를 조회할 수 있다.
- 강의/강좌와 문제 세트 연결을 운영자가 관리할 수 있다.
- GCS 공개 URL 의존을 줄이고 Private bucket, Signed URL, 인증 프록시 중 적절한 보안 방식을 확정한다.

## 2026-06-11 변경사항

- 정답 처리된 문제의 재제출을 동기 검증하고 `409 SUB-006`으로 차단한다.
- 문제세트 결과, 제출 결과 상세, 관리자 학생 제출 조회의 제출 코드 필드를 `submittedCode`로 통일했다.
- 챗봇 FastAPI 요청은 외부 계약 호환성을 위해 JSON 키 `submitted_answer`를 유지하고 내부 모델명만 `submittedCode`를 사용한다.
- 사용하지 않는 `TextSubmission`, `AnswerGradingService`, 주관식 제출 필드 의존을 제거했다.
- Python Runner가 Signed URL 데이터셋을 임시 디렉터리에 다운로드하고 `DATASET_PATH`로 사용자 코드에 전달한 뒤 실행 종료 시 삭제하도록 검증했다.
- 프론트는 문제 상태가 `CORRECT`이면 제출 버튼을 비활성화하고 기존 제출 코드는 `submittedCode`로 표시해야 한다.
