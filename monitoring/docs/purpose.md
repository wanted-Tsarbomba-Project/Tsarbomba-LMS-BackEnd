# 이 모니터링은 무엇을 위한 것인가 — 두 용도와 지금의 범위

> 읽기 전 1줄: 같은 스택(Prometheus·Grafana·Loki)을 쓰지만 **목적이 둘**이다.
> **이번에 우리가 하는 건 ① 성능 개선이다.** ②는 운영 배포 때 확장한다.

---

## 두 용도

모니터링은 성격이 다른 두 가지에 쓰인다. 헷갈리면 "내가 부하를 만드냐 / 사용자가 만든 걸 지켜보냐"로 가른다.

| | **① 성능 개선** | **② 오류 모니터링** |
|---|---|---|
| 답하는 질문 | "느린가? 어디가 병목인가?" | "실패하나? 왜 실패하나?" |
| 성격 | **일회성 실험** (내가 부하를 만들어 측정) | **상시 운영 관측** (사용자가 만든 실패를 지켜봄) |
| 환경 | loadtest 프로파일 + 도커 MySQL(3307) + k6 = **격리된 실험실** | **운영(prod)** + 실 트래픽 + 실 DB |
| 시간축 | 실험 구간(분 단위) | 상시(일·주) + 알림 |
| 핵심 지표 | p95, 커스텀 **Timer**, RPS, Hikari | **에러율(4xx/5xx)**, 실패 사유별 **Counter(reason)**, 에러 로그 |
| 주력 도구 | k6 + Timer + 전후 비교 | Counter(reason) + Loki 로그 + traceId + **알림** |
| 결과물 | "인덱스로 408ms → 8ms" 같은 개선 증명 | "결제 5xx 급증 → 알림 → traceId로 원인 추적" |

핵심 차이 한 줄:
```
① 성능 = 내가 부하를 만들어 본다 (실험실, loadtest)
② 오류 = 사용자가 만든 실패를 지켜본다 (운영, prod)
```
→ 그래서 **②는 운영 환경에 붙어야 의미**가 있다. 지금 도커(loadtest) 스택에서 ②를 하면 "내가 k6로 만든 가짜 에러"만 보게 된다.

## 지금 우리의 범위 = ① 성능 개선

이번 과제의 본질은 **"부하테스트로 병목을 찾고 개선을 증명하는 것"** = ①이다. 그래서:

- 모든 문서·프로세스(7단계)·k6·시드·전후 비교는 **① 기준**으로 본다.
- 메트릭을 심을 때 우선순위는 **"느린 구간을 재는 Timer"** (성능용).
- 환경은 **loadtest 프로파일 + 도커 MySQL**. 운영 DB(RDS)에 부하 금지.

> 실행 순서·방법은 [`domains/README.md`](domains/README.md), 기술 선택 기준은 [`monitoring-guide.md`](monitoring-guide.md).

## ② 오류 모니터링 — 토대만 있고, 운영 배포 때 확장 (지금은 안 함)

②에 쓰일 토대는 이미 깔려 있다(Phase 0 공통):
- `domain` 태그로 에러율 쿼리 가능: `...{domain="...",status=~"4..|5.."}`
- `MdcLoggingFilter` 로그 + `traceId`, Loki `level="ERROR"` / `event=api_error` 추적

하지만 **②를 제대로 하려면 아래가 더 필요**하다 (지금은 보류, 운영 들어갈 때):

| 빠진 것 | 내용 |
|---|---|
| 도메인별 실패 Counter | `<도메인>_xxx_failed_total{reason=...}` 가 아직 안 심김 (성능 Timer만 있음) |
| 운영 환경 수집 | 현재 스택은 loadtest 로컬 전용 → 실사용자 에러를 못 봄. prod actuator/Loki 수집 구성 필요 |
| 알림 | Grafana Alert / Alertmanager 미구성 (`monitoring-guide.md` §11은 예시만) |

> 즉 **메트릭을 심을 때 "성능용 Timer"가 먼저고, "오류용 Counter(reason)"는 ② 확장 시 추가**한다. 다만 지금 도메인 코드를 만질 때 실패 지점이 명확하면 `event=<도메인>_xxx_failed reason=...` 로그 한 줄 정도는 미리 남겨두면, 나중에 ② 확장이 공짜로 따라온다.

## 한 줄 결론

- **지금**: ① 성능 개선 (부하테스트 → 병목 → 최적화 증명). 환경은 loadtest.
- **나중(운영)**: ② 오류 모니터링 (에러율·사유·알림). 토대는 깔려 있으니 Counter·운영수집·알림만 얹으면 된다.
